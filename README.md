CurriculumKG contains the ontology, source data, generated documentation, and the Python triplification script used to materialize curriculum data into RDF.

### Repository Structure
* `schema/` - Directory containing the CurrKG ontology design files (e.g., schema diagrams).
* `script/` - Triplification workflow and data assets.
  * `data/` - Directory with source data for CurrKG.
  * `materialization/` - Directory containing the materialized instance of CurrKG (`.ttl` file) generated by the scripts.
* `documentation/` - Working notes and supporting documentation for ontology design decisions.
* `docs/` - Published/generated ontology website assets, serializations, provenance output, and WebVOWL viewer files.

### Python Materialization Script
The main script is [script/updated-triplification.py](/Users/chrisdavisj/curriculum-kg/script/updated-triplification.py). It:
- reads a CSV input
- creates or extends an RDFLib graph
- maps curriculum, module, persona, learning path, media, author, event, and topic data into CurriculumKG triples
- writes Turtle output

Run it from inside the `script/` directory:

```bash
python updated-triplification.py \
  --input-data ./data/data-okg.csv \
  --current-file ./materialization/new-schema-currkg.ttl \
  --output ./materialization/new-schema-currkg.ttl
```
